Here’s a “Vision Initiation Pack” that communicates Oblivion-meets-NASA-meets-Apple clearly without talking about tech. The theme is: show the experience, name the rules, and make it reviewable.

1. Experience Vision One-Pager

The fast alignment doc: what it feels like, what it’s for, why it matters.
Include:
• The north-star sentence (“You are inside an infinite operational workspace…”)
• 3 pillars (e.g., Calm Power / Living Telemetry / Spatial Continuity)
• 6 “non-negotiables” (desktop-first, morph-not-pages, etc.)
• 3 screenshots/mood frames + 1 diagram (the “space map” teaser)

This is basically the “one pager” pattern used for quick stakeholder alignment. ￼

2. UX Vision Statement

A slightly more formal, durable version of the vision that survives team changes.
Include:
• Vision statement + “how we’ll know we’re there”
• What the system will never do (anti-goals)
• What users can confidently expect every time

NN/g has a strong guide for building these collaboratively and research-backed. ￼

3. Design Principles Manifesto (1–2 pages)

This is your “we can break rules because we have rules” doc.
Include 8–12 principles, each with:
• Principle name (e.g., “Reveal through motion, not menus”)
• Do / Don’t examples
• A measurable “passes/fails” test

Design principles are explicitly meant to define quality and anchor critique. ￼

4. Spatial Model Map (The “Infinite Workspace Geography”)

A single diagram that makes the “I’m in the space” concept instantly legible.
Include:
• Hub at center + “territories” for Tarva apps
• How you navigate (pan/zoom + return-to-hub)
• Minimap/radar concept
• What changes with zoom levels (macro → meso → micro)

If you later need IA rigor, keep IA separate from any sitemap-style artifacts. ￼

5. Storyboard + Walkthrough Script (5–15 frames)

This is the clearest way to share morphing behavior without specs.
Include:
• Login “secret reveal”
• Landing in hub
• Selecting an app tile → morph into app cluster
• Returning to hub
• A “receipt moment” (proof/trace)

The storyboard method is a known alignment tool for prototyping decisions. ￼

6. “Living Details” Catalog (Micro-Details Library)

A menu of ambient behaviors so everyone builds the same vibe.
Structure it like a pattern library:
• Ambient (always-on subtle)
• Reactive (responds to events)
• Rituals (scan → focus → confirm → receipt)
For each item: purpose, intensity, frequency, and where it’s allowed (background vs foreground).

(Framing this as documentation is consistent with good UX design documentation practices.) ￼

7. Telemetry & Status Language Guide

This prevents “random dashboard soup” and makes the hub feel mission-control credible.
Include:
• Status states (healthy / degraded / down / unknown)
• What each state means + what users do next
• What’s ambient vs actionable
• Alert severity vocabulary and visual hierarchy

Service-blueprint thinking helps when telemetry reflects real backstage systems. ￼

8. Service Blueprint (Backstage Reality Map)

Not for customers—this is internal, so it’s perfect.
Include:
• Frontstage: what user sees in the hub
• Backstage: what systems generate telemetry/events
• Support processes: identity, audit/receipts, notifications, logs

Service blueprints explicitly connect touchpoints to people/process/systems. ￼

9. Critique Rubric (Definition of “Feels Right”)

A lightweight scoring sheet to keep taste consistent:
• Calmness (no anxious motion)
• Readability at distance
• Spatial continuity (no page-y jumps)
• Ritual clarity (user always knows “what just happened”)
• “Alive” details never block primary tasks

(Principles → critique is a standard pairing.) ￼

10. Pilot “Golden Path” Scenarios (3 short scripts)

Three mini narratives you can demo anytime: 1. Check system health in 10 seconds 2. Jump into TarvaCORE and confirm action → receipt 3. Investigate an alert → see evidence → return to hub

User journeys/flows are useful tools for mapping goal completion paths. ￼
